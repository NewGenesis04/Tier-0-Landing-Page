Overview

CMS Tier0 is the last step in the chain of services responsible for new data acquisition and processing in the CMS Collaboration.

Tier0 is responsible for the creation and permanent storage of CMS RAW data, which is the basis of all scientific results for the collaboration. All other data representations are transient with a short lifetime. This is the most critical task that Tier0 is responsible for.

In the order of priority Tier0 team is responsible for the following tasks:

  1. Streamer data repacking into RAW format and safe data storage on tape at two different sites. Till it happens we must always have two independent copies to prevent data lost in case of a storage failure.

  2. Data repacking must be performed fast enough to avoid a situation when we have just one copy of the streamer files because buffers at P5 are full. In an extreme case CMS may be forced to stop data taking if we run out of space in the EOS streamer storage area.
    Besides data storage, repacking is also important as a test of data readability and if it is not readable this needs to be investigated and fixed ASAP to prevent data loss.

  3. Express processing is critical for the detector calibration and it is expected to be available shortly after data taking (within a few hours). It provides detailed information on the data quality.

  4. Prompt reconstruction is the first attempt to reconstruct all new data completely. It is often used for urgent physics studies and publications. Beside that role, prompt reconstruction ensures that the RAW data is usable. All other processes use streamer files and if RAW is not tested we may end up with corrupted unusable data even if streamer data is Ok.
